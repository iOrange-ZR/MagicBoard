# Penguin Magic 魔改版 v0.2.0 更新说明

**版本**：v0.2.0  
**基于**：魔改 v0.1.0 + 主项目画布与 API 能力  
**说明**：本版本主要新增可灵 O1（Kling Omni）视频生成支持，包括多图/视频输入、占位符引用、输入预览与插入、以及参考视频 URL 校验与友好提示。

---

## 本次更新概要

### 可灵 O1 视频生成

- **模型入口**：视频节点模型选择中增加「可灵 O1」，使用 Omni 接口 `POST .../kling/v1/videos/omni-video`，支持多模态输入。
- **输入规则**：至多 **7 张图片**，或 **1 个视频 + 4 张图片**；输入顺序按连接顺序排列，新连接的素材排在后面。
- **UI 参数**（参考图 1 布局，无 VIP/特殊配色）：
  - **输入节点**：列表展示上游图片/视频节点名称及缩略图预览。
  - **生成模式**：720p / 1080p（对应 API mode std / pro）。
  - **生成时长**：3s～10s 可选。
  - **比例**：智能 / 9:16 / 1:1 / 16:9。
- **提示词占位符**：支持在提示词中使用 `<<<image_1>>>`、`<<<video_1>>>` 等引用第 N 张图或第 N 个视频；输入列表每行提供「插入」按钮，在提示词光标处插入对应占位符。
- **可灵 2.6**：仍保留原有「时长 5/10/自动」与「开启音频」；O1 与 2.6 在视频节点内按所选模型切换不同 UI。

### 参考视频与 URL 校验

- **问题**：可灵 API 要求参考视频通过 URL 由服务器拉取，且 `video_url` 不能为空；本地路径、`data:`/`blob:`、localhost 等地址服务器无法访问，会报错（如 `get_ratios_error` / `media_duration error`）。
- **处理**：
  - **调用前校验**：若当前为可灵 O1 且存在参考视频输入，但视频 URL 不可被服务器访问（本地、data、blob、localhost 等），则**不发起 API 请求**，并在节点上提示错误。
  - **提示文案**：`当前为本地或不可访问的视频地址，无法作为参考视频。视频上传功能服务器尚在开发中，请暂时使用可公网访问的视频 URL，或先断开视频输入仅用图片生成。`
  - **输入列表提示**：当某条视频为本地/不可访问时，在该行显示「（本地地址无法使用，将阻止生成；视频上传功能服务器开发中）」。
- **API 错误映射**：当接口返回 `get_ratios_error` 或 `media_duration`/`audio duration` 相关错误时，前端展示上述友好说明，便于用户理解原因。

### 其他调整

- **可灵 O1 负向提示词**：O1 节点不再展示负向提示词输入框（API 未使用）；可灵 2.6 仍保留。
- **可灵 O1 生成数量**：移除「生成数量」1/2/3/4 的 UI 与逻辑，批量数量与其它模型一致，由用户触发时的数量决定。

## 技术变更摘要

| 类型       | 说明 |
|------------|------|
| 类型定义   | `pebblingTypes`：`KlingO1InputItem`，`NodeData` 增加 `klingResolution`、`klingAspectRatio`，`klingDuration` 扩展为 3～10 及 auto |
| 服务层     | `klingVideoService`：`createKlingO1Task`、`getKlingOmniTaskStatus`、`waitForKlingOmniCompletion`；`isKlingServerAccessibleVideoUrl`；O1 错误码/文案映射 |
| 画布解析   | `index`：`resolveInputsForKlingO1`（按连接顺序返回 images、videos、items），O1 分支校验与调用 |
| 画布 UI    | `CanvasNode`：可灵 O1 输入列表（预览 + 插入）、生成模式/时长/比例；本地视频行提示；`insertKlingPlaceholder`、`klingPromptRef` |
| 执行逻辑   | 批量与单节点执行中：O1 时使用 `resolveInputsForKlingO1`；参考视频 URL 不可用时提前拦截并提示，不调用 API |

## 使用说明

- **可灵 O1**：在视频节点选择「可灵 O1」，连接图片或视频节点作为输入；在提示词中可使用 `<<<image_1>>>`、`<<<video_1>>>` 等，或通过输入列表的「插入」在光标处插入。
- **参考视频**：若使用参考视频，须为**可公网访问的 MP4/MOV URL**；本地文件、`/files/`、data 链接等会触发校验并阻止调用，并提示「视频上传功能服务器开发中」。
- **仅用图片**：无需参考视频时，断开视频输入即可，仅图片 + 提示词会正常提交。
